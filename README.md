# AI Resume Analyzer & Job Portal

An intelligent full-stack AI-powered Job Portal and Resume Analyzer that streamlines the hiring process for both Recruiters and Students. The application leverages Google Gemini and OpenAI APIs to automatically analyze resumes, rank applicants, and conduct interactive AI Mock Interviews tailored to job descriptions.

---

## 🚀 Key Features

### 👨‍🎓 For Candidates / Students
- **Smart Job Search**: Browse, search, filter, and save job postings.
- **AI Resume Analysis**: Upload your resume (PDF/Doc) during application and receive an instant compatibility match score and detailed feedback relative to the job requirements.
- **AI-Powered Mock Interviews**: Take dynamic, real-time mock interviews generated by AI (Gemini/OpenAI) based on the job's target tech stack and requirements, complete with audio/text feedback.
- **Application Tracking**: Monitor the status of your applications (Pending, Shortlisted, Rejected).
- **Profile & Resume Management**: Manage your skills, experience, projects, and primary resumes.
- **Real-Time Notifications**: Get notified when you are shortlisted or invited to interview.

### 💼 For Recruiters
- **Job Lifecycle Management**: Post, update, and manage job openings easily.
- **AI Applicant Ranking & Scoring**: View applicants sorted automatically by their AI-calculated resume match score. Save hours of manual screening.
- **Rich Analytics & Dashboards**: View recruiter analytics (total jobs, application rates, top skills in demand, and recruitment funnel charts).
- **Applicant Assessment**: Deep-dive into student profile details, resume match metrics, and scheduled interviews.

## 🛠️ Technology Stack

| Layer | Technologies Used |
|---|---|
| **Frontend** | React (Vite), TailwindCSS, Zustand (State Management), Framer Motion (Animations), Recharts (Charts/Analytics), React Router DOM, React Icons, Lucide React |
| **Backend** | Node.js, Express.js, MongoDB (Mongoose), Multer & Multer Storage Cloudinary (File Uploads), Cookie Parser, Helmet (Security), Morgan (Logging) |
| **AI Integration** | Google Generative AI (Gemini SDK), OpenAI API |
| **Email & Authentication** | Nodemailer (OTP / Email verification), JSON Web Tokens (JWT), Google OAuth 2.0 (`@react-oauth/google` & `google-auth-library`) |
| **Deployment** | Vercel (both Frontend & Backend) |

## ⚙️ Environment Variables Setup

Create a `.env` file in the **backend** directory:

```env
PORT=5000
MONGODB_URI=your_mongodb_connection_string
JWT_SECRET=your_jwt_signing_key
NODE_ENV=development

# Frontend URL for CORS
FRONTEND_URL=http://localhost:5173

# Nodemailer setup for Email verification & OTPs
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_app_password

# Cloudinary Setup for Resumes / Profile Photos
CLOUDINARY_CLOUD_NAME=your_cloudinary_cloud_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret

# AI Models Configuration
GEMINI_API_KEY=your_google_gemini_api_key
OPENAI_API_KEY=your_openai_api_key

# Google OAuth Setup
GOOGLE_CLIENT_ID=your_google_oauth_client_id
```

Create a `.env` file in the **frontend** directory:

```env
VITE_API_BASE_URL=http://localhost:5000/api/v1
VITE_GOOGLE_CLIENT_ID=your_google_oauth_client_id
```

## 🏃 Getting Started

### Prerequisites
- Node.js installed on your system (v18.x or above recommended)
- MongoDB database (local instance or MongoDB Atlas cluster)

### Steps

1. **Clone the repository:**
   ```bash
   git clone https://github.com/sheetanshumohan/AI-Job-Portal.git
   cd AI-Job-Portal
   ```

2. **Setup the Backend:**
   ```bash
   cd backend
   npm install
   # Add environment variables in .env file
   npm run dev
   ```

3. **Setup the Frontend:**
   ```bash
   cd ../frontend
   npm install
   # Add environment variables in .env file
   npm run dev
   ```

4. Open your browser and navigate to `http://localhost:5173` to interact with the web app!

## 🌐 Deployment (Vercel)

Both frontend and backend are configured to deploy seamlessly on Vercel:
- **Backend Server**: Uses `vercel.json` to routing incoming API queries to `src/app.js` using `@vercel/node`.
- **Frontend SPA**: Uses `vercel.json` to rewrite all client paths to `/index.html` to support React Router.
